# Инструкция по запуску приложения "Склад"

## 📋 Перед запуском

### 1. Проверьте что есть:
- Папка с проектом (все файлы из репозитория)
- Папка `nginx-1.28.3` с файлом `nginx.exe`
- Файл `.env` с настроенными параметрами

### 2. Проверьте файл `.env`

Откройте `.env` в блокноте и убедитесь что:

```env
JWT_SECRET=ваш-уникальный-секрет-минимум-32-символа
ONEC_PASSWORD=ваш-пароль-от-1C
```

⚠️ **ВАЖНО:** Замените `JWT_SECRET` на случайную строку! Текущее значение небезопасно.

---

## 🚀 Запуск (один клик)

1. Откройте папку с проектом
2. Дважды кликните на **`start.bat`**
3. Дождитесь сообщения "ГОТОВО!"
4. Откроются два окна (не закрывайте их!)

---

## ✓ Проверка работы

1. Откройте браузер
2. Введите: `http://localhost:8080/`
3. Должна загрузиться страница входа

**Или с другого компьютера:**
- `http://IP_СЕРВЕРА:8080/`

---

## 🔧 Диагностика

### Если приложение не работает:

1. **Запустите `check.bat`** — покажет статус всех компонентов

2. **Посмотрите логи:**
   - Backend: окно "Sklad Backend" — там выводы
   - Nginx: `nginx-1.28.3\logs\error.log`

3. **Проверьте что процессы запущены:**
   - Откройте Диспетчер задач
   - Должны быть `node.exe` и `nginx.exe`

### Частые проблемы:

| Проблема | Решение |
|----------|---------|
| Порт 8080 занят | Закройте другие программы (Skype, другой nginx) |
| Порт 8000 занят | Закройте другие node процессы |
| Node.js не найден | Установите с https://nodejs.org/ |
| nginx не найден | Скачайте portable nginx в папку проекта |
| 404 при открытии | Проверьте что папка `dist` создана после сборки |

---

## 🛑 Остановка

1. **Правильный способ:** Запустите **`stop.bat`**
2. **Или вручную:** Закройте окна "Sklad Backend" и "Sklad Nginx"

---

## 🔄 Обновление

1. Остановите приложение (`stop.bat`)
2. Скопируйте новые файлы в папку (замените старые)
3. Запустите `start.bat` заново

---

## 📞 Адреса приложения

| Место | Адрес |
|-------|-------|
| На сервере | `http://localhost:8080/` |
| С другого ПК | `http://IP_СЕРВЕРА:8080/` |
| Backend API | `http://IP_СЕРВЕРА:8000/sklad/api/` |

---

## 🔐 Безопасность

1. **Смените JWT_SECRET** в `.env` перед запуском
2. **Проверьте брандмауэр** — откройте порт 8080 для сети
3. **Не открывайте порт** в интернет (только локальная сеть)

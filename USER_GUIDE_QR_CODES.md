# Инструкция по Использованию Новой Системы QR Кодов

## Быстрый Старт

### 1. Запуск Сервера
```bash
cd /home/alex/sklad/sklad
node server.js
```

Сервер стартует на `http://localhost:8000/sklad/api`

### 2. Генерирование QR Кодов для Заказа

**В UI:**
1. Перейдите на страницу Заказы
2. Откройте нужный заказ (клик на строку)
3. На вкладке "Позиции заказа" появится прогресс-бар с текущим количеством QR кодов
4. Нажмите кнопку "Генерировать QR коды" (если она появится)
5. Укажите:
   - Количество кодов
   - Товар
   - Количество кодов
6. Коды будут сгенерированы и сохранены на бэкенде

**Через API:**
```bash
curl -X POST http://localhost:8000/sklad/api/orders/xxx-xxx/qr-codes/generate \
  -H "Content-Type: application/json" \
  -d '{
    "quantity": 5,
    "productId": "prod-123",
    "productName": "Красная мебель",
    "generatedBy": "admin",
    "employeeId": "emp-123",
    "employeeName": "John Doe"
  }'
```

### 3. Просмотр QR Кодов

**В UI:**
1. Откройте заказ
2. В таблице позиций видите количество готовых QR кодов
3. Прогресс-бар показывает процент завершения (отсканировано / всего)

**Через API:**
```bash
curl http://localhost:8000/sklad/api/orders/xxx-xxx/qr-codes
```

Ответ будет содержать список всех QR кодов заказа с их статусами.

### 4. Сканирование QR Кода

**Через API:**
```bash
curl -X POST http://localhost:8000/sklad/api/qr-codes/qr-123/scan \
  -H "Content-Type: application/json" \
  -d '{
    "newStatus": "scanned",
    "employeeId": "emp-123",
    "employeeName": "John Doe"
  }'
```

Статус кода изменится на `scanned` и будет залогирован.

### 5. Редактирование Окраски

**В UI:**
1. Откройте заказ
2. В правой панели найдите блок "Окраска"
3. Нажмите "Редактировать"
4. Введите информацию об окраске (например "Красная глянцевая")
5. Нажмите "Сохранить"

Это действие будет залогировано как `order_painting_updated`.

### 6. Просмотр Истории Операций Сотрудника

**В UI:**
1. Перейдите на страницу Сотрудники
2. Кликните на сотрудника для открытия деталей
3. Нажмите на строку сотрудника чтобы открыть более подробный вид
4. Перейдите на вкладку "История операций"
5. Видите последние 10 операций сотрудника в обратном хронологическом порядке
6. Каждая операция показывает:
   - Тип операции (с иконкой)
   - Дату и время
   - Связанный заказ
   - Связанный товар
   - QR код (если применимо)
   - Дополнительные детали

**Через API:**
```bash
curl "http://localhost:8000/sklad/api/employees/emp-123/operations?limit=10"
```

### 7. Полный Просмотр Логов

**Через API с фильтрацией:**
```bash
# Все операции по генерированию QR кодов
curl "http://localhost:8000/sklad/api/operation-logs?type=qr_code_generated&limit=100"

# Все операции конкретного сотрудника
curl "http://localhost:8000/sklad/api/operation-logs?employee=emp-123&limit=50"

# Все операции конкретного заказа
curl "http://localhost:8000/sklad/api/operation-logs?order=order-123&limit=50"

# Статистика
curl "http://localhost:8000/sklad/api/operation-logs/stats"
```

## Типы Логируемых Операций

| Операция | Когда логируется | Данные |
|----------|------------------|--------|
| `qr_code_generated` | При создании QR кодов | quantity, product, order |
| `qr_code_scanned` | При сканировании QR кода | status, employee |
| `qr_code_deleted` | При удалении QR кода | qr_code, order, product |
| `order_painting_updated` | При изменении окраски | new_value, order |

## Статусы QR Кодов

- **generated** - Только создан, не использован
- **scanned** - Отсканирован один раз
- **shipped** - Отгружен
- **deleted** - Удален

## Важные Моменты

### Где хранятся QR коды?
- На бэкенде в таблице `local_qr_codes` (SQLite)
- Персистентны (не теряются при перезагрузке)
- Глобально уникальны

### Где хранится история операций?
- На бэкенде в таблице `operation_logs` (SQLite)
- Полная история всех действий с временем и сотрудником
- Быстрый поиск благодаря индексам

### Синхронизация с 1С
- QR коды остаются локально, не отправляются в 1С
- Окраска синхронизируется в поле `onec_orders.painting`
- Логи операций сохраняются локально для аудита

### Производительность
- Индексы по order_id, code, status для QR кодов
- Индексы по employee, operation_type, created_at для логов
- Быстрый поиск даже с большим объемом данных

## Примеры Реальных Сценариев

### Сценарий 1: Производство товара

```
1. Сотрудник открывает заказ
   → Log: заказ открыт (если добавим)

2. Сотрудник генерирует 10 QR кодов для товара A
   → Log: qr_code_generated x10
   → В БД: 10 новых QR кодов со статусом "generated"

3. При производстве каждый QR код отсканируется
   → Log: qr_code_scanned x10
   → В БД: QR коды обновлены со статусом "scanned"

4. История операций сотрудника показывает:
   - 10 операций сканирования
   - 1 операция генерирования
   - Всего 11 операций за сеанс
```

### Сценарий 2: Редактирование окраски

```
1. Сотрудник откры заказ
2. Нажимает "Редактировать" на окраске
3. Вводит "Красная матовая"
4. Нажимает "Сохранить"
   → API: POST /onec/orders/painting
   → Log: order_painting_updated { newValue: "Красная матовая" }
   → В БД: поле painting заказа обновлено
   → История операций: 1 новая операция
```

### Сценарий 3: Аудит действий

Начальник хочет посмотреть какие QR коды сгенерировал сотрудник Иван за день:

```bash
curl "http://localhost:8000/sklad/api/operation-logs?type=qr_code_generated&employee=ivan&limit=100"
```

Ответ покажет:
- Когда были сгенерированы коды
- Для какого заказа
- Для какого товара
- Сколько кодов

## Возможные Проблемы и Решения

### Q: Почему не видны QR коды после перезагрузки?
A: **Старая система была в localStorage.** Новые коды хранятся на бэкенде. Если вы создали коды **до** этого обновления, они были в localStorage и потеряны. Создайте новые коды - они будут персистентны.

### Q: Как восстановить удаленный QR код?
A: В таблице operation_logs есть запись о удалении с полной информацией. Можно вручную вставить в local_qr_codes и изменить статус обратно.

### Q: Какой объем данных может хранить?
A: SQLite хорошо работает с миллионами записей. Для средних объемов (десятки тысяч операций) никаких проблем.

### Q: Как экспортировать логи?
A: Через SQL query:
```sql
SELECT * FROM operation_logs 
WHERE operation_type = 'qr_code_generated'
AND created_at > datetime('now', '-7 days')
ORDER BY created_at DESC
```

## Команды для Администратора

### Очистить все QR коды заказа
```bash
sqlite3 .data/app.db "DELETE FROM local_qr_codes WHERE order_id = 'xxx-xxx'"
```

### Посмотреть статистику операций
```bash
sqlite3 .data/app.db "
SELECT operation_type, COUNT(*) as count, COUNT(DISTINCT employee_id) as employees
FROM operation_logs
GROUP BY operation_type
ORDER BY count DESC
"
```

### Архивировать старые логи (старше 90 дней)
```bash
sqlite3 .data/app.db "DELETE FROM operation_logs WHERE created_at < datetime('now', '-90 days')"
```

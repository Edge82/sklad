## Роли и права доступа

| Роль | Описание | Доступные страницы |
|------|----------|-------------------|
| **admin** | Администратор системы | Все |
| **director** | Директор | Все, включая Сотрудников и Отчёты |
| **manager** | Менеджер | Все, включая Сотрудников и Отчёты |
| **storekeeper** | Кладовщик | Склад, Сканирование, Инструменты, Профиль, Заказы, Перемещения |
| **worker** | Рабочий | Склад (просмотр), Мои инструменты, Профиль |

**Тестовые пользователи** (создаются автоматически при первом запуске):

| Логин | Пароль | Роль |
|-------|--------|------|
| `admin` | `admin` | admin |
| `manager` | `manager` | manager |
| `storekeeper` | `storekeeper` | storekeeper |
| `worker` | `worker` | worker |

## Страницы приложения

### Вход / Логин

- **URL:** `/login`
- **Доступ:** все неавторизованные пользователи

Форма авторизации с полями «Имя пользователя» и «Пароль». После входа перенаправляет на страницу Склада.

---

### Склад (Inventory)

- **URL:** `/inventory`
- **Доступ:** все роли (кроме worker — только просмотр)

Основная страница со списком всех товаров и материалов. Возможности:

- **Поиск:** по названию, артикулу (SKU), штрихкоду
- **Фильтры:** по категории, складу, статусу
- **QR-коды:** генерация QR-кода для каждого товара
- **Просмотр остатков:** текущее количество на складах
- **Просмотр карточки товара:** детальная информация, единица измерения, цена, фото

*Кладовщик:* может редактировать количество, создавать приходные/расходные накладные.

---

### Готовая продукция

- **URL:** `/inventory/finished` или `/finished-products`
- **Доступ:** storekeeper, manager, admin, director

Таблица готовой продукции с остатками. Возможности:

- **Приём готовой продукции:** кнопка «Принять готовую продукцию» — форма с выбором номенклатуры и указанием количества. Продукция поступает на склад «Готовая продукция».
- **Поиск и фильтрация:** по названию, артикулу, дате

---

### Заказы (Orders)

- **URL:** `/orders`
- **Доступ:** storekeeper, manager, admin, director

Управление заказами покупателей, синхронизированными из 1С. Возможности:

- **Просмотр списка заказов:** номер, дата, клиент, статус, сумма
- **Детали заказа:** состав заказа (товары, количество, цена), статус, дата отгрузки
- **QR-коды:** генерация QR-кодов для товаров в заказе (для маркировки при сборке)
- **Печать QR-кодов:** печать этикеток с QR-кодами
- **Фильтры:** по статусу, дате, клиенту

Статусы заказов: «Новый», «В работе», «Собран», «Отгружен», «Завершён».

---

### Сканирование (Scan)

- **URL:** `/scan`
- **Доступ:** storekeeper, worker

Сканирование штрихкодов / QR-кодов товаров для сборки заказов. Возможности:

- **Сканирование товара:** ввод штрихкода или QR-кода в поле ввода (поддерживаются ручные сканеры)
- **Подтверждение сборки:** после сканирования всех товаров заказа — кнопка «Завершить сборку»
- **История сканирований:** просмотр отсканированных товаров в текущей сессии

---

### Инструменты (Tools)

- **URL:** `/tools`
- **Доступ:** storekeeper, manager, admin, director

Учёт инструментов и оборудования. Возможности:

- **Список инструментов:** название, категория, статус, местоположение
- **Поиск:** по названию, QR-коду, штрихкоду, местоположению
  - *Транслитерация:* при вводе русских букв в раскладке QWERTY (например, «qwerty» вместо «йцукен») поиск автоматически преобразует их
- **QR-коды:** генерация QR-кода для каждого инструмента
- **Выдача / возврат:** отметка о выдаче инструмента сотруднику и возврате
- **Поломки:** регистрация поломки инструмента, просмотр истории поломок
- **Фильтры:** по статусу («В наличии», «Выдан», «На ремонте»), по категории

---

### Профиль (Profile)

- **URL:** `/profile`
- **Доступ:** все авторизованные пользователи

Информация о текущем пользователе. Возможности:

- **Имя и роль:** отображение ФИО и роли
- **Смена пароля:** форма смены пароля
- **Последние операции:** список последних 5 операций пользователя (создание заказов, сканирования, выдача инструментов и т.д.)
- **Смена фото профиля** (если поддерживается)

---

### Мои инструменты в работе

- **URL:** `/profile/tools`
- **Доступ:** worker, а также все роли

Список инструментов, выданных текущему пользователю. Возможности:

- **Просмотр:** название, дата выдачи, планируемая дата возврата
- **Сканирование QR:** отметка о возврате инструмента через сканирование QR-кода

---

### Сотрудники (Employees)

- **URL:** `/employees`
- **Доступ:** только director, manager

Управление сотрудниками предприятия. Возможности:

- **Список сотрудников:** ФИО, должность, отдел, контакты
- **Добавление сотрудника:** форма создания
- **Редактирование:** изменение данных сотрудника
- **Карточка сотрудника:** при переходе на `/employees/:id` — детальная информация, история операций с инструментами

---

### Движение материалов (Shipment)

- **URL:** `/shipment`
- **Доступ:** storekeeper, manager, admin, director

Учёт отпуска материалов в производство / перемещений. Возможности:

- **Создание накладной:** выбор материалов, количества, склада-получателя, сотрудника
- **Список накладных:** дата, номер, статус, ответственный
- **Печать накладной:** формирование печатной формы

---

### Заказы на перемещение (Transfer Orders)

- **URL:** `/transfer-orders`
- **Доступ:** storekeeper, manager, admin, director

Создание и управление заказами на перемещение товаров между складами. Возможности:

- **Список заказов:** номер, дата, склад-отправитель, склад-получатель, статус
- **Фильтры:** по статусу («В работе», «Завершён»), по дате
- **Детали заказа:** состав, количество, статус сканирования
- **Создание нового заказа:**
  1. Выбор склада-отправителя и склада-получателя из выпадающего списка
  2. Сканирование штрихкодов товаров для добавления в заказ
  3. Корректировка количества в таблице
  4. Отправка в 1С — заказ создаётся сразу со статусом «Завершён»
  5. При недоступности 1С — сохраняется локально с префиксом `LOCAL-`
- **Сборка заказа:** сканирование товаров по заказу со статусом «В работе»
- **Завершение заказа:** отправка отметки о завершении в 1С

---

### Интеграция (Integration)

- **URL:** `/integration`
- **Доступ:** admin, manager, director

Настройки и статус интеграции с 1С. Возможности:

- **Просмотр статуса:** соединение с 1С (доступна / недоступна)
- **Последняя синхронизация:** дата и время последней успешной синхронизации
- **Ручная синхронизация:** кнопка «Синхронизировать» для принудительной синхронизации всех данных (номенклатура, остатки, заказы, единицы измерения и т.д.)
- **Лог синхронизации:** просмотр результатов последних синхронизаций

---

### Отчёты (Reports)

- **URL:** `/reports`
- **Доступ:** только director, manager

Аналитические отчёты и статистика. Возможности:

- **Отчёт по остаткам:** текущие остатки товаров на складах
- **Отчёт по движениям:** приход/расход материалов за период
- **Отчёт по сотрудникам:** количество операций по каждому сотруднику
- **Отчёт по заказам:** статистика по заказам (количество, сумма, статусы)
- **Экспорт:** выгрузка отчётов (PDF / Excel)

## Частые сценарии

**Я кладовщик, мне нужно переместить товар со склада А на склад Б:**
1. Откройте «Заказы на перемещение» (`/transfer-orders`)
2. Нажмите «Новый заказ на перемещение»
3. Выберите склад-отправитель и склад-получатель
4. Отсканируйте штрихкоды товаров, укажите количество
5. Нажмите «Сохранить и отправить в 1С»
6. Заказ создастся со статусом «Завершён»

**Мне нужно собрать заказ покупателя:**
1. Откройте «Сканирование» (`/scan`)
2. Отсканируйте QR-код заказа (или выберите из списка)
3. Сканируйте каждый товар из заказа
4. Когда все товары собраны, нажмите «Завершить»

**Нужно выдать инструмент сотруднику:**
1. Откройте «Инструменты» (`/tools`)
2. Найдите инструмент по названию или QR-коду
3. Нажмите «Выдать», выберите сотрудника
4. После возврата — нажмите «Принять»

**1С временно недоступна, что делать?**
- При создании заказа на перемещение он сохранится локально с префиксом `LOCAL-`
- При следующей успешной синхронизации заказы автоматически отправятся в 1С
- Статус соединения можно проверить на странице «Интеграция» (`/integration`)
